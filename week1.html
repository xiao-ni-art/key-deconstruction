Week 01 / Object Deconstruction

第一周：以钥匙为主题进行思维发散。

第一周的任务不是马上做产品，而是先把“钥匙”从日常印象里拿出来。它看起来只是一个开门工具，但其实包含结构、材料、工艺、动作、情绪、身份和场景。

Assignment Translation

我把第一周理解为：先建立一把钥匙的“底层数据库”。

01

结构数据库

手柄决定握持方式，匙身决定插入方向，牙花决定匹配逻辑，挂孔决定携带方式。钥匙的每一处形态都有功能理由。

02

材料数据库

金属带来冷硬、安全和耐久；塑料带来轻量、颜色和低成本；柔性材料能改变触感和保护方式。

03

动作数据库

插入、旋转、听见咔嗒声、推门进入，这一连串动作形成了钥匙的仪式感，也给后续互动产品提供了线索。

04

情绪数据库

钥匙让人产生安全感、归属感和拥有感。它小到可以放进口袋，却代表一个空间是否属于我。

Early Product Clues

第一周形成的是“产品线索”，不是最终方案。

A

握持与身份

从钥匙手柄出发，思考可定制的握持形态。不同颜色、纹理和轮廓可以让钥匙成为个人身份的随身标记。

提取元素
手柄 / 触感 / 挂孔
产品线索
个性化挂件、识别钥匙套
B

牙花与匹配

钥匙的牙花本质上是一种匹配规则。它可以转译成拼插、卡扣、关卡和模块化玩具的结构逻辑。

提取元素
牙花 / 插入 / 对齐
产品线索
解锁玩具、教学机关
C

旋转与反馈

开锁时的旋转动作和声音反馈很有仪式感，可以转译成灯光、声音、按钮或学习任务的触发方式。

提取元素
旋转 / 咔嗒声 / 触发
产品线索
互动装置、知识钥匙
D

携带与收纳

钥匙天然具有随身属性，它可以连接收纳、挂载和防丢功能，为后续折叠盒、钥匙盒和模块容器提供基础。

提取元素
挂孔 / 随身 / 防丢
产品线索
便携容器、模块收纳

Process Engine

从物件到概念，我用了五步拆解。

Step 01

日常物件：钥匙

从手柄、匙身、牙花、挂孔、材料与开锁动作进入观察，保留它作为“入口”的核心身份。

Domain Map

我把钥匙拆成七个可以继续连接的域。

KEY 打开 / 解锁 / 释放

Domain 01

结构域：手柄、匙身、牙花、挂孔

钥匙的轮廓不是装饰，它决定握持、识别、携带和插入动作。结构拆开后可以转译成模块、轨道、拼插和游戏关卡。
